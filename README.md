# Расчёт справедливой стоимости участия в прибыли для паевого договора страхования жизни

Курсовая работа (ВМК МГУ, кафедра исследования операций).
Расчёт справедливой стоимости участия в прибыли для unit-linked договора
страхования жизни с инвестированием в строящуюся недвижимость методом
**глубокого хеджирования (Deep Hedging)** на неполном рынке.

## Что внутри

| Файл | Описание |
|------|----------|
| `calculator.ipynb` | Итоговый расчётный ноутбук: загрузка данных → калибровка → модель → расчёт стоимости участия → анализ чувствительности и стресс-тесты |
| `realty_index.csv` | Ряд индексов цен первичного рынка жилья РФ (Росстат, 2021Q1–2026Q1), собранный из квартальных публикаций |

## Краткое описание метода

Базовый актив (строящаяся недвижимость) неторгуем, а риски незавершения
строительства и смертности нехеджируемы — рынок неполный, и классическая
risk-neutral оценка не даёт единственной цены. Стоимость обязательства
определяется как минимальный капитал, при котором остаточный риск позиции
(по энтропийной мере риска) неположителен. Оптимальная стратегия
хеджирования аппроксимируется нейронной сетью; справедливая стоимость
участия вычисляется как разность цен договора с участием и без него:
**V_part = π(ξ_α) − π(ξ_0)**.

Волатильность недвижимости калибруется по данным Росстата с устранением
сглаживания (десглаживание Гелтнера), что повышает оценку волатильности
примерно вдвое.

## Требования

- Python 3.10–3.12 (64-bit) — **не 3.13**, под него на момент написания нет готовых колёс PyTorch
- Видеокарта NVIDIA с CUDA — желательно (ускоряет обучение); код работает и на CPU, но медленнее

## Установка и запуск

### 1. Клонировать репозиторий

```bash
git clone https://github.com/Limon4ik1LoVe/calculation_cost_participation_in_profits.git
cd calculation_cost_participation_in_profits
```

### 2. Создать виртуальное окружение

```bash
python -m venv venv
```

Активировать:
- Windows (PowerShell): `venv\Scripts\Activate.ps1`
- Windows (cmd): `venv\Scripts\activate.bat`
- Linux/macOS: `source venv/bin/activate`

### 3. Установить PyTorch

**С поддержкой GPU (NVIDIA, CUDA 12.1):**
```bash
pip install torch --index-url https://download.pytorch.org/whl/cu121
```
Если эта версия CUDA не подойдёт, подберите команду под свою систему на
https://pytorch.org/get-started/locally/ (варианты cu118 / cu121 / cu124).

**Только CPU (без видеокарты):**
```bash
pip install torch
```

### 4. Установить остальные зависимости

```bash
pip install -r requirements.txt
```

### 5. Проверить, что GPU виден (опционально)

```bash
python -c "import torch; print(torch.cuda.is_available())"
```
`True` — расчёт пойдёт на видеокарте; `False` — на процессоре.

### 6. Запустить ноутбук

```bash
jupyter notebook calculator.ipynb
```
Запускайте ячейки по порядку (Run All). Первая ячейка печатает
используемое устройство (`cuda` или `cpu`).

> **Важно:** файл `realty_index.csv` должен лежать в той же папке, что и
> ноутбук, и Jupyter должен быть запущен из этой папки. Иначе загрузчик не
> найдёт данные и переключится на синтетический ряд (о чём сообщит в выводе).

## Параметры

Все параметры модели вынесены в отдельную ячейку конфигурации (раздел 2
ноутбука). Для анализа чувствительности достаточно менять одно значение и
перезапускать соответствующие ячейки. На GPU можно увеличить число
траекторий `M` и число эпох `EPOCHS` для более точных и гладких результатов.

## Источники данных

- Индексы цен на первичном рынке жилья: Росстат, раздел «Цены»
- Статистика проблемных объектов (для калибровки интенсивности
  незавершения): реестр проблемных объектов, ЕРЗ.РФ / ДОМ.РФ

## Структура расчёта

1. Загрузка данных и устранение сглаживания (Гелтнер)
2. Калибровка параметров (рынок, риск незавершения, контракт)
3. Полная модель договора (скачок незавершения, базисный риск, смертность, два режима премии)
4. Верификация (Блэк–Шоулз и Monte-Carlo)
5. Расчёт справедливой стоимости участия
6. Анализ чувствительности и стресс-тесты
